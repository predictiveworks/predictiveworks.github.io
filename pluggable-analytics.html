Pluggable Analytics

Motivation

In a business context, data analytics has to answer the following questions:

  • What has happened?
  • Why did it happen?
  • What will happen?
  • How can we make it happen?

Answering these clear-cut questions nowadays requires to adopt a plethora of different solutions with human data and software experts in the loop to stick all pieces individually together.

Many excellent open source big data analytics & computing libraries exist, but each with a certain focus or lens on the data universe, its own infrastructure and technology.

An integrated data analytics platform that seamlessly covers all flavors of modern data analytics seems to be an utopia for data-driven enterprises.

Features

Pluggable Analytics can be best characterized by its features:

  • The foundation is a cloud-ready pipeline platform with standardized containers to build pluggable components from data integration to machine learning and more.

  • Plugins are built on top of standardized containers and represent purpose-built component instances, say connectors to SaaS applications (CRM, ERP, SCM, HRM etc.), or, demand prediction for resource planning and more.

  • Business users can organize and orchestrate plugins as data pipelines leveraging a graphical editor without the need of programming skills.

  • Building data pipelines to transform enterprise raw data into business insights & foresights becomes as easy as defining a business project with an approved project management tool.

Implementation

PredictiveWorks. leverages the huge standardization and unification capabilities of two open-source giants: Apache Spark and Google CDAP.

A 3-Step-Approach is applied to transform any 3rd party analytics library written in Java or Scala into a set of ready-to-use pluggable pipeline components.
